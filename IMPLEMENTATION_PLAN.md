## 二、业务流程

```
[关键词挖掘] ──→ [内容创作] ──→ [SEO优化] ──→ [发布] ──→ [推广] ──→ [变现]
     ↑                                                        ↓
     └──────────────── [数据反馈] ←──────────────────────────────┘
```

### Phase 1: 冷启动（第1-4周）
- **SEO分析师** 每日输出 5 个长尾关键词
- **主编** 审核关键词，排优先级（RICE打分）
- **日文编辑** 每日 1 篇对比文章（2000+字，50+ h2/h3）
- **开发工程师** 自动化：RSS→小红书、Twitter 自动发

### Phase 2: 加速期（第2-3月）
- **社区运营** Yahoo知恵袋 每日3答，附带链接
- **SEO分析师** 追踪排名，调整关键词策略
- **主编** 写工厂对比特辑（VSF/Clean/PPF 横评）
- **数据分析** 首月流量报告，调整内容方向

### Phase 3: 变现期（第4-6月）
- AdSense + A8.net 联盟上线
- UGC 板块运营，社区自增长
- 每月"本月高仿特征汇总"热点文章

## 三、成本应对策略（正式版涨价）

### 3.1 当前 DeepSeek API 价格

### 3.2 预计正式版涨价后
| 模型 | 预计输入 | 预计输出 |
|------|---------|---------|
| deepseek-v4-flash | ¥5/百万token | ¥20/百万token |
| deepseek-v4-pro | ¥40/百万token | ¥160/百万token |

### 3.3 降本策略

```
1. 分级使用
   ├── 主编（深度内容，占比20%）→ v4-pro 不降级
   ├── 日文编辑（批量生产，占比50%）→ 涨价后改用 v4-flash
   ├── SEO/运营（数据处理，占比25%）→ 永远用 v4-flash
   └── 分析/报告（无需推理）→ 用最便宜的模型

2. 缓存策略
   ├── 已审核文章模板 → 直接复用，不重复调API
   ├── 关键词库 → 预生成，每周更新一次
   └── 对比表格数据 → 一次生成，多页引用

3. 批量策略
   ├── 每周一批处理5篇，减少单次调用次数
   ├── 凌晨低峰调用，避开高峰期限流
   └── 本地缓存 translation（日文术语库）

4. 混合策略
   ├── 初稿：v4-flash 便宜批量出稿
   ├── 审核：主编（人）快速过一遍
   ├── 终稿：v4-pro 精修最重要3-5篇
   └── 翻译：deepl API 或本地模型（日文翻译便宜）

5. 备选方案
   ├── OpenRouter 聚合调用，找最便宜的路由
   ├── 买 DeepSeek 充值套餐（预付折扣）
   └── 本地部署 ollama + 开源日文模型（零成本）
```

### 3.4 月成本预算

|------|------|--------|--------|
| 保守（30篇/月） | ¥1,550 | ¥7,750 | ¥2,500 |
